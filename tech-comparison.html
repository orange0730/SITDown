🏆 最佳推薦：Vanilla JS + GSAP

基於你的 SITDown 專案現狀，我推薦採用 Vanilla JS + GSAP 的漸進式增強方案。這個選擇能在保留現有程式碼基礎上，大幅提升動畫品質和開發效率。

✅ 優點

  • 保留現有程式碼，無需重寫
  • GSAP 是業界動畫標準，文檔完善
  • 性能優異，硬體加速
  • 漸進式整合，風險低
  • 動畫時間軸控制精準
  • 豐富的緩動函數庫

❌ 缺點

  • 增加約 60KB 檔案大小
  • 商業使用需要授權（但有免費版）
  • 需要學習 GSAP API
  • 沒有框架的組件化優勢

實作範例：增強版貓爪動畫

// 1. 在 index.html 中引入 GSAP
<script src="https://cdnjs.cloudflare.com/ajax/libs/gsap/3.12.2/gsap.min.js"></script>

// 2. 增強現有的貓爪動畫
const catPawTL = gsap.timeline({paused: true});

catPawTL
  .to('#cat-paw', {
    y: -300,
    rotation: -10,
    duration: 0.8,
    ease: 'power2.out'
  })
  .to('.claw', {
    scale: 1.5,
    rotation: (i) => [-20, 0, 20][i],
    stagger: 0.05,
    ease: 'back.out(1.5)'
  }, 0.2)
  .to('.homepage-content', {
    filter: 'blur(5px)',
    opacity: 0.7,
    duration: 0.6
  }, 0);

// 3. 拖曳時更新進度
function onDrag(progress) {
  catPawTL.progress(progress);}
